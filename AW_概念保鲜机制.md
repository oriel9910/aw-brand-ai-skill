> **版本**：v2.0（从 v1.0「概念保鲜机制」重写）
> **更新日期**：2026-04-20
> **定位**：解决品牌语言"重复滥用导致疲劳"的根本问题。核心原则：**案例只做引导不做调用，每次生成必须从原则出发生成全新表达。**

## 一、核心原则：永远生成，永不调用

### 1.1 一条硬规则

> **所有案例、范例、入库句，仅供理解方向。AI 在生成文案时，必须从品牌态度 + 方法论 + 具体语境出发，生成全新表达。不得直接复制、调用、复用任何已有案例。**

### 1.2 为什么？

| 旧方式（调用式） | 新方式（生成式） |
|-------------|-------------|
| 查素材库 → 找到匹配的句子 → 复制/微调 | 读品牌态度 → 选方法论 → 理解语境 → 从零生成 |
| 句子越用越腻 | 每次都是新的 |
| 需要手动管理"用了几次" | 不需要管理，因为从不复用 |
| AI 变成"搬运工" | AI 变成"理解品味后原创的人" |

### 1.3 什么可以直接引用，什么不行？

| 层级 | 内容 | 使用方式 |
|------|------|---------|
| **品牌态度/原则/定义** | "鼓励出发""有趣>有用""不走性价比"等 | ✅ **直接引用**，作为硬约束 |
| **品牌口号** | "让每一次出行都成为勇敢探索" | ✅ **直接引用**，这是永久资产本身 |
| **方法论框架** | 6大手法、海报三层、类型A~F | ✅ **直接引用**，作为思考路径 |
| **方法论洞察** | 素材库中的"为什么选这条"分析 | ✅ **参考方向**，理解用户偏好 |
| **所有案例/范例/入选句** | 任何具体的文案表达 | ❌ **只做引导**，不得直接调用 |

**判断标准**：如果它是"态度"或"规则"，可以反复引用。如果它是"某一句具体的表达"，永远只能参考，不能复制。

## 二、AI 生成时的思考路径

### 2.1 标准生成流程

```
① 读品牌态度（共通基石）
   → "AW 相信什么？反对什么？"

② 选方法论（6大手法 + 叙事章节）
   → "用哪个手法？讲哪个章节的故事？"

③ 读决策洞察（素材库中的"为什么选这条"）
   → "用户在这个手法下偏好什么方向？排斥什么方向？"

④ 理解具体语境
   → "什么渠道？什么产品？什么时机？什么人群？"

⑤ 从零生成
   → 基于以上四层理解，写出全新表达
   → 不得复制任何已有案例
```

### 2.2 关键区别

| 步骤 | 旧方式 | 新方式 |
|------|--------|--------|
| ③读素材库 | 找到匹配的句子，准备调用 | 读"为什么选那条"的洞察，理解偏好方向 |
| ⑤生成 | 在已有句子上微调/替换词语 | 从态度+方法论+洞察出发，写全新表达 |

## 三、语义场：AI 的思考维度清单

> 语义场不是"素材库"，不是用来"挑句子"的。它是 AI 在生成时的**思考维度清单**——提醒 AI 可以从哪些角度切入，避免每次都想到同一个方向。

### 3.1 使用方式

| ❌ 错误用法 | ✅ 正确用法 |
|-----------|-----------|
| 从语义场里挑一个表达直接用 | 看一眼语义场的维度，然后从零生成新表达 |
| "车顶空着"用过了，换成"车库里的车落灰" | "空间闲置"这个维度我上次用了，这次换"时间浪费"维度 |

### 3.2 核心语义场（思考维度，非素材库）

#### 维度①：未出发的状态（为什么还没出门？）

| 子维度 | 思考方向（不是现成句子） |
|--------|---------------------|
| 空间闲置 | 车顶/后备箱/车库/车位的"空"——还有什么具象的"空"可以写？ |
| 时间浪费 | 假期/周末/下班后的"没出门"——还有什么时间锚点可以制造反差？ |
| 心理拖延 | "下次一定""还没准备好"——还有什么拖延行为可以具象化？ |
| 社交落差 | 别人在外面 vs 你在家——还有什么社交场景可以制造对比？ |
| 身体困住 | 城市里的"出不去"——还有什么身体感受可以对比城市vs山野？ |

#### 维度②：出发的瞬间（从"不动"到"动"的转折）

| 子维度 | 思考方向 |
|--------|---------|
| 装备动作 | 装/拓展/变形/解锁——还有什么动作可以描述"改变"？ |
| 出发动作 | 走/出发/踩油门——还有什么动作可以描述"离开"？ |
| 心理转折 | 不再等了/不需要理由——还有什么心理变化可以描述？ |
| 物理变化 | 车变高了/车顶满了——还有什么物理变化可以描述？ |

#### 维度③：在路上的体验（出发之后感受到什么？）

| 子维度 | 思考方向 |
|--------|---------|
| 视觉 | 星空/落日/山脊线——还有什么视觉画面可以描述？ |
| 听觉 | 风声/引擎声/安静——还有什么声音可以描述？ |
| 触觉 | 温度变化/风——还有什么触觉可以描述？ |
| 嗅觉 | 泥土/松树/雨后——还有什么气味可以描述？ |
| 时间感 | 天黑了不想回/不看表——还有什么时间感受可以描述？ |
| 社交 | 不回消息/朋友圈不发——还有什么社交行为可以描述？ |

#### 维度④：回来之后的余韵（出发之后带回了什么？）

| 子维度 | 思考方向 |
|--------|---------|
| 身份变化 | "车顶有东西的人"——还有什么身份标签可以描述？ |
| 对比反差 | 地库vs旷野/城市vs山野——还有什么对比可以制造？ |
| 期待延续 | 下次去哪/还想装什么——还有什么期待可以描述？ |

### 3.3 防重复思考规则

每次生成前，AI 自检：

| # | 自检问题 | 作用 |
|---|---------|------|
| 1 | 上次我用了哪个语义子维度？ | 避免连续用同一个维度 |
| 2 | 上次我用了哪个感官？ | 避免连续用视觉（最常用的感官） |
| 3 | 上次我从谁的视角写的？ | 避免连续用品牌视角 |
| 4 | 当前有什么时事/季节/场景可以借势？ | 用即时语境生成，而非翻旧表达 |

## 四、方法论是活的：随迭代进化

### 4.1 核心理念

> **方法论不是静态规则，而是随着每次迭代不断精炼的"活的理解"。**

现在的"矛盾对撞"定义是：
> 两句对称，前一句"对的"，后一句"错的"或反转的，形成价值判断。

经过多轮迭代后，它应该进化成：
> 矛盾对撞：两句对称，前一句对的，后一句反转。
> - AW 用户偏好"否定局限+打开可能"方向（而非"制造愧疚"方向）
> - 句式上偏好"不该只有/不只是/不限于"结构
> - 长度上偏好 ≤12 字
> - 语气上偏好开放感而非判断感
> - 锚点上偏好"空间/时间/社交"维度，避免"愧疚/浪费"维度

### 4.2 迭代分析原则：用力写选中的，温柔带没选的

> **选中的才是信号，没选的只是噪音。洞察从"选了什么"中提炼，不从"没选什么"中积累排斥清单。**

#### 为什么？

旧的分析模式：选了的简单带过，没选的用一个标签"打死"（说教/太土/太AI），然后写进"避免"清单。

这会导致两个问题：
1. **用户不敢选**——觉得"不选=这个词被判死刑"
2. **洞察库越来越窄**——每条"避免"都在缩小可用的表达空间，最后AI什么都不敢写

#### 正确的分析方式

| 对象 | 分析力度 | 原则 |
|------|---------|------|
| **选中的** | **用力分析** | 为什么这条就是比其他的好？好在哪里？好到什么程度？逐字拆解 |
| **没选的** | **温柔带过** | 不是"不行"，只是"这次不是它"。一句话说明可能的原因，但不定性为"禁止" |

#### 洞察卡片的写法变化

```markdown
# ❌ 旧方式（用力写没选的）
- 用户没选："别让十月的风，只在手机里"
  - 原因：说教
  - 生成指导：❌ 避免"别让…"句式

# ✅ 新方式（用力写选中的）
- 用户选了："十月的风，比老板的消息温柔"
  - 为什么好：拿"老板的消息"开了所有人都懂的玩笑，但不攻击任何人。
    "温柔"把触觉变成了情感判断。12字刚刚好。
  - 生成指导：✅ 用日常共鸣制造幽默；✅ 感官锚点定义情感
```

#### 没选的怎么处理？

没选的只在洞察卡片中**一句话带过**，格式为：

```markdown
- 未选："xxx"（这次不是它）
```

不写原因分析，不写"避免"，不写"禁止"。它只是"这次没被选中"，不代表"永远不能用"。

#### 什么时候可以从"没选"中提炼洞察？

只有一种情况：**同一种表达方式连续3次以上都没被选中**。这时可以谨慎地写一条"偏好方向"（不是"排斥方向"），措辞为：

```markdown
- 观察：连续3次迭代中，"别让…"句式均未被选中
- 可能的偏好：用户可能更倾向"邀请式"而非"否定式"开头
- 注意：这不是禁止，只是当前阶段的观察，可能随语境变化
```

### 4.3 方法论进化的来源

方法论进化的原材料来自**每次迭代中"选中的"决策洞察**——用户选了什么、为什么。

这些洞察沉淀在《文案素材库》中（详见该文档的新结构），随着积累越来越细腻，方法论也就越来越精准。

### 4.4 进化触发机制

| 触发条件 | 动作 |
|---------|------|
| 同一个手法连续3次迭代，用户都选了同一类方向 | 更新该手法的"偏好方向"描述 |
| 同一种表达方式连续3次以上未被选中 | 谨慎记录一条"观察"（不是"排斥"） |
| 出现全新的用户偏好（之前没见过的选择模式） | 在该手法下新增一条洞察 |
| 方法论洞察积累到5条以上 | 考虑更新《品牌文案参考规则》中该手法的正式定义 |

## 五、案例的使用边界

### 5.1 案例的唯一正确用法

案例存在的意义是**让 AI 理解"这种风格是对的"**，而不是**让 AI 复制"这句话可以用"**。

| 场景 | 正确用法 | 错误用法 |
|------|---------|---------|
| AI 第一次学习 AW 风格 | 读10条入选句，理解"AW 长什么样" | 记住这10条，下次直接用 |
| AI 写新文案前 | 回顾相关洞察，理解"用户在这个手法下偏好什么" | 找到最匹配的一条，微调后输出 |
| 用户问"AW 是什么风格" | 用案例作为示例解释风格 | 把案例当成"标准答案" |

### 5.2 案例的三个层级

| 层级 | 内容 | AI 的使用方式 |
|------|------|-------------|
| **品牌级案例** | "让每一次出行都成为勇敢探索" | 理解品牌精神，可在品牌故事中引用 |
| **方法论级案例** | "去见山海，别陷在人海" | 理解"去vs别"手法的标准形态，但生成时必须写新的 |
| **迭代级案例** | 素材库中的入选句 | 只读"为什么选这条"的洞察，不读句子本身 |
