# AW Product：产品与参数查询

## 面向同事的标准回复（优先）

当用户问 **「XX车型能不能装」「这个产品什么参数」「适配哪些车」「价格多少」「这个车的人群画像」** 时：

我是 **AW 产品数据库**助手：我可以帮你查车型适配、产品参数、SKU、价格、人群画像。

你只需要告诉我**产品名和/或车型**，我就能查到对应信息。

---

## 权威与主流程（Agent 执行时用）

1. **先读** [AW_品牌共通基石.md](../../../AW_品牌共通基石.md) — 品牌态度、产品心智。
2. **再按「L3 检索协议」读** [AW_产品参数与SKU.md](../../../AW_产品参数与SKU.md) — **禁止**无锚点整文件加载（见下节）。
3. 按下方查询规范输出结果。

---

## L3 检索协议（硬约束 · 省 token）

与本仓库技能分层一致：**L1** YAML 元数据、**L2** 本 `SKILL.md`、**L3** 产品真源大表。L3 **体积大**，必须按片段取数。

| 禁止 | 必须 |
|------|------|
| 无关键词就 `read_file` 通读整份 `AW_产品参数与SKU.md` | 先读真源内 **「〇、检索索引」**，再定检索词 |
| 凭记忆或训练数据填写适配/价格/尺寸/SKU | 命中段落内原文为准；缺失写「真源未录入」 |
| 同一对话里对同一查询重复全文扫描 | 已读过的段落可在回复中保留结论，增量只读新锚点 |

**推荐执行顺序**：

1. 用 `rg` / `grep -n` 在仓库根（或 skill 安装目录，与软链同级）对 `AW_产品参数与SKU.md` 搜 **车型 / 产品线 / 章节标题关键词**（多试同义词，见真源「〇、检索索引」）。  
2. 根据行号 **只读窗口**（例如上下各 40 行），或 `sed`/分段 `read_file`；单轮任务累计读取 L3 建议 **不超过约 200～400 行**，仍不足再扩大窗口。  
3. 输出时附 **「数据出处：真源第〇/×章 + 检索词」** 一句（内部可追溯即可，不必给用户看行号）。

## 俗称解析与模糊检索（硬规则）

真源 **`AW_产品参数与SKU.md` →「〇.1 内部俗称、代号 ↔ 真源用语」** 为权威对照；**禁止**只用用户原词做一次字面检索就下结论。

| 用户说法（示例） | 应先映射为 | 再检索 |
|----------------|------------|--------|
| M款、m款、M平台、mlinkx | **M'LinkX** | `M'LinkX|MLinkX|mlinkx|## 三、|### 4.2` |
| S款、s款、入门 storm | **Storm** | `Storm|## 二、|### 4.1` |
| 与车型连写的 **S2**、`S2/专用`、`S2/AIR` | **多为 M'LinkX 拓展件支架代号**，≠ 口语「S款」 | `S2|专用支架|侧挂|### 3.` |

**消歧输出（简洁优先）**：

1. **首句给判定**：用一句话写清「按内部习惯，你说的 S款 我按 **Storm**（或 M款→**M'LinkX**）理解」。  
2. **若 Storm 章无该车型行**：不要冗长辩解；直接给出「第二章当前无 [车型] 行」+ **若指 M'LinkX**，跳转第三章/§4.2 已命中内容（如豹8与 `S2/专用支架` 同现的表）。  
3. **禁止**：因未命中「S款」二字就回答「真源无 S款」；须先展开俗称再搜。  
4. **禁止**：把 **S2 代号**与 **S款=Storm** 混为一谈而不向用户说明。

**检索展开示例**（豹8 + 俗称）：`rg -n "豹8|方程豹|Storm|M'LinkX|S2" AW_产品参数与SKU.md`，再按章节拆读。

## 支持的查询类型

### 查询①：车型适配查询

**输入**：车型名（如"豹8""问界M9""Model Y"）

**输出格式**：

```
## [车型名] 适配查询结果

### 适配状态
| 产品线 | 适配状态 | 备注 |
|--------|---------|------|
| M'LinkX | ✅ 支持 | — |
| Storm | ✅ 支持 | — |
| 无级悬停天幕 | ✅ 通用 | 不绑定车型 |
| ... | ... | ... |

### 推荐搭配
[基于人群画像推荐的产品组合]

### ⚠️ 提示
以上数据来自产品数据库，请以内部系统为准。
```

### 查询②：产品参数查询

**输入**：产品名（如"M'LinkX""无级悬停天幕"）

**输出格式**：

```
## [产品名] 参数

| 参数项 | 数值 |
|--------|------|
| 材质 | ... |
| 重量 | ... |
| ... | ... |
```

### 查询③：价格/SKU查询

**输入**：产品名 + 车型（如适用）

**输出格式**：

```
## [产品名] 价格与SKU

| SKU | 适配车型 | 价格 | 备注 |
|-----|---------|------|------|
| ... | ... | ... | ... |
```

### 查询④：人群画像查询

**输入**：车型名 或 人群类型（如"女性""年轻人""家庭奶爸"）

**输出格式**：

```
## [车型/人群] 人群画像

| 维度 | 内容 |
|------|------|
| 核心人群 | ... |
| 人群特征 | ... |
| 推荐产品 | ... |
| 文案方向 | ... |
```

### 查询⑤：卖点口径查询

**输入**：产品名

**输出格式**：

```
## [产品名] 卖点口径

| 维度 | 口径 |
|------|------|
| 一句话卖点 | ... |
| 核心卖点 | ... |
| 情感价值 | ... |
| 适合人群 | ... |
```

### 查询⑥：反向查询——按产品找适配车型

**输入**：产品名（如"Storm"）

**输出格式**：

```
## Storm 适配车型清单

### 按品牌分类
- 奥迪：A3/A4/A5/A6/Q3/Q5（✅），Q7（⚠️）
- 宝马：1系/2系/3系/4系/5系/X1/X3（✅），X5（⚠️）
- ...

### 统计
| 统计项 | 数量 |
|--------|------|
| 覆盖品牌 | ... |
| 覆盖车型 | ... |
| 完全支持 | ... |
| 部分支持 | ... |
| 不支持 | ... |
```

## 被其他技能调用的接口规范

其他技能通过 `bash` 命令按需读取数据源文件，接口约定如下：

### 接口1：查适配

```bash
# 先在仓库根或本 skill 安装目录执行；先定位行号再扩大上下文
rg -n "豹8|方程豹" AW_产品参数与SKU.md
# 再按需：grep -n "豹8" AW_产品参数与SKU.md | head
```

### 接口2：查参数

```bash
rg -n "^## 四、|M'LinkX|Storm" AW_产品参数与SKU.md
# 确认小节边界后再 sed 截取，避免跨章误读
```

### 接口3：查人群

```bash
rg -n "^## 六、|豹8|人群" AW_产品参数与SKU.md
```

### 接口4：查卖点

```bash
rg -n "^## 五、|一句话卖点|M'LinkX" AW_产品参数与SKU.md
```

## 数据缺失时的处理

当查询的数据在真源中标注为"待补充"或字段为空时：

1. **如实告知**用户该数据暂未录入
2. **建议用户提供**数据源（Excel/CSV/截图），我来补充
3. **不要编造数据**——这是硬约束
