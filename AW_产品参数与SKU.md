## 〇、检索索引（Agent 必读 · L3 按需加载）

**本文件禁止整份无目标通读。** 先用关键词命中章节，再只读对应片段（`rg` / `grep` 或按标题区间 `read_file` 带行范围）。

| 需求 | 优先检索关键词 / 章节锚点 | 对应大章 |
|------|---------------------------|---------|
| 产品线一览、轨道 A/B/C | `产品线总览`、表格「类型」 | **一** |
| Storm 某车型适配 | 车型名、`## 二、Storm`、品牌小标题（如 `### 2.4 奥迪`） | **二** |
| M'LinkX 某车型适配 | 车型名、`## 三、M'LinkX`、`方程豹`、`豹8` | **三** |
| 尺寸 / 材质 / 载重等硬参数 | 产品名、`## 四、各产品线`、小节标题 | **四** |
| 一句话卖点 / 卖点口径 | 产品名、`## 五、各产品线` | **五** |
| 人群画像、文案方向 | 车型或人群、`## 六、车型人群画像` | **六** |
| SKU / 价格 | `## 七、价格体系`、SKU、产品名 | **七** |

**车型同义词（检索时多试几个词）**：如「豹8」「方程豹 8」「方程豹8」「豹 8」等，以表中正式地名为准交叉命中。

### 〇.1 内部俗称、代号 ↔ 真源用语（Agent 必须先展开再检索）

同事口语与表格**字面不一定一致**。检索前把俗称**映射到产品线 + 展开一组正则关键词**，避免「只搜 S款 搜不到就判无数据」。

| 同事口语 / 简写 | 真源中指什么 | 检索时务必展开的 `rg` 关键词（示例） |
|----------------|-------------|----------------------------------|
| **M款**、M线、M平台、ML、mlinkx、MLINKX | **M'LinkX** 车顶拓展平台 | `M'LinkX|MLinkX|mlinkx|## 三、` |
| **S款**、S线、入门平台、Storm、storm | **Storm** 车顶平台（第二章） | `Storm|## 二、|车顶平台` |
| **S2**、S2支架、`S2/专用`、`S2/AIR`、`S2/HEAVY`（与车型同现） | **M'LinkX 拓展件侧挂/支架体系代号**（见第三章 P1 侧挂箱、攀梯等表），**不是**「S款=Storm」里的 S | `S2|专用支架|AIR支架|HEAVY支架|侧挂` |
| 标准款 / AIR款 / HEAVY款 / BASE 02 | M'LinkX **平台款式**（§3.2），与 Storm 的「支架形式」不同维度 | `标准款|AIR款|HEAVY款|BASE 02|### 3.2` |

**消歧（高频踩坑）**：

1. **「S款」≠ 字面搜 `S2`**：口语 S款 多指 **Storm**；真源里的 **S2** 多指 **M'LinkX 配件代际**。用户说「豹8 S款平台」→ 先按 **Storm + 豹8** 查 **第二章**；若该车型在 Storm 章**无行**，再并列给出 **M'LinkX 章**里豹8相关行（含 `S2/专用支架` 等），并一句话说明二者区别。  
2. **「M款平台参数」**→ 查 **§4.2 M'LinkX 平台**（整平台参数）+ 若有车型再查 **第三章** 该车行。  
3. **「S款平台参数」**→ 查 **§4.1 Storm 平台**（整平台参数）+ 若有车型再查 **第二章** 该车行。

**标准动作（示例）**：

1. 含俗称时：`rg -n "豹8|方程豹|Storm|M'LinkX|S2" AW_产品参数与SKU.md` → 按命中所在大章（二/三/四）分段只读。  
2. 已知在大章内时，用 `## 二、`、`## 三、` 等标题或 `rg '^## '` 定位后分段读取。
