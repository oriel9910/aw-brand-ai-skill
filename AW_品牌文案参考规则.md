## 四、文案方法论

> 以下方法论从小红书顶级户外文案中提炼，是AW品牌文案创作的核心手法库。

### 手法① 矛盾对撞

用一个字制造张力，让读者停下来。

**结构**：两句对称，前一句"对的"，后一句"错的"或反转的，形成价值判断。

**范例**：


| 原文                         | 矛盾对撞     |
| -------------------------- | -------- |
| 最**轻**的背包里，背着最**重**的决心     | 轻 vs 重   |
| 去感受时间的**温度**，别去追赶时间的**速度** | 温度 vs 速度 |
| 去见**山海**，别陷在**人海**         | 山海 vs 人海 |
| 装得下**旷野**，也进得去**地库**       | 旷野 vs 地库 |
| **停下来**，才是**出发**           | 停下 vs 出发 |


**AW应用**：

- "去出发，别去路过"
- "没有目的地的时候，(哪里都是)"
- "不赶路的时候，(才在路上)"

---

### 手法② 去vs别的句式

不直接说"你应该怎样"，而是同时说"去XX"和"别XX"，让读者自己得出结论。

**结构**：去…，别…。先给方向，再划边界，态度鲜明但不说教。

**范例**：


| 原文       | 去…   | 别…     |
| -------- | ---- | ------ |
| 去感受时间的温度 | 去感受  | 别去追赶   |
| 去野       | 去野   | 别去YEAH |
| 去见山海     | 去见山海 | 别陷在人海  |


**AW应用**：

- "去山野，别去人海"
- "去装满，别去将就"
- "去感受风，别去追赶风"

---

### 手法③ 具象化抽象情绪

把"焦虑""压力""疲惫"这些抽象词，变成可以用身体感知的画面。

**结构**：不说"你很累"，说"皱巴巴的心事"；不说"别装"，说"别把快乐的姿势变成拍照的手势"。

**范例**：


| 抽象情绪   | 具象化表达          |
| ------ | -------------- |
| 焦虑/心事  | 皱巴巴的心事         |
| 职场倦怠   | 写字楼里没有的新鲜感     |
| 社交表演   | 别把快乐的姿势变成拍照的手势 |
| 被安排的自由 | 别把自由的方向变成行程的转向 |
| 虚假的假期  | 别让五一变成沙发到床的距离  |


**AW应用**：

- "别把山野，存进收藏夹"
- "后备箱关不上的，是自由"
- "皱巴巴的假期，展开就平了"

---

### 手法④ 括号制造"私语感"

用括号把关键词单独拎出来，像是在跟读者说悄悄话。

**结构**：正文 + (括号里的点睛之笔)。括号里的词是整句话要读者带走的那句话。

**范例**：


| 原文                | 括号内容 | 效果        |
| ----------------- | ---- | --------- |
| 背着**(最重)**的决心     | 最重   | 强调反差      |
| **(让风)**把皱巴巴的心事吹平 | 让风   | 强调自然的力量   |
| 哪里都可以是**(方向)**    | 方向   | 强调自由      |
| 往哪走都是**(往前走)**    | 往前走  | 强调每一步都有意义 |


**AW应用**：

- "没有目的地的时候，(哪里都是)"
- "不赶路的时候，(才在路上)"
- "地图上没有的路，(走过了就有了)"
- "车顶空着，(可惜了)"

---

### 手法⑤ 一语双关

一个词同时承载两层含义，读一遍有一层意思，读两遍有更深的。

**范例**：


| 原文                | 双关               |
| ----------------- | ---------------- |
| 去**野**，别去**YEAH** | 野=自然 / YEAH=社交假嗨 |
| "焕"身硬壳            | 焕=焕新 / 换=换上      |
| 她们**野**行          | 野=户外 / 也=也敢于     |
| 来这里**自然**会有       | 自然=大自然 / 自然=自然而然 |


**AW应用**：

- "出发，自然有答案"（自然=大自然/自然而然）
- "轻装上阵，(重)在出发"（重=重量轻/重在行动）
- "展开假期，(收起)焦虑"（展开/收起=天幕动作/心理切换）

---

### 手法⑥ 极致短句

所有顶级文案都在10~15个字以内解决战斗，没有一个逗号以上的长句。

**原则**：能断句就断句，能砍字就砍字，留白比填满更有力量。

**范例**：


| 原文         | 字数  |
| ---------- | --- |
| 去野，别去YEAH  | 6字  |
| 去见山海，别陷在人海 | 10字 |
| 写字楼里没有的新鲜感 | 10字 |
| 山从不规训我     | 6字  |


**AW应用**：

- "去出发" / "就现在"
- "装上，走"
- "停。看。"

## 五、海报文案结构规范

### 5.1 标准三层结构


| 层级       | 功能           | 字数建议   | 注意事项                              |
| -------- | ------------ | ------ | --------------------------------- |
| **视觉大字** | 场景向往，触发"我想去" | 6~15字  | 不提产品，只描绘向往的画面；用上述6种手法创作           |
| **副标题**  | 品牌态度，承接向往    | 8~15字  | 用品牌自有语言体系，自然带出AW；可以有态度但不攻击        |
| **底部说明** | 利益支撑，给抓手     | 3~4个词组 | 斜杠分隔；不重复品牌名/产品名；给经销商和消费者一个具体的信息抓手 |


### 5.2 底部说明原则

- ✅ 用利益点（"全系装备 / 适配100+车型 / 全国100+授权门店"）
- ✅ 用最轻量的信息给抓手
- ❌ 不重复品牌名和产品名（已在视觉大字或副标题中出现）
- ❌ 不堆砌卖点参数

### 5.3 底层潜台词

整张海报的底层逻辑：**你该出门了 → 出门就该有AW → AW让这一切更好**。但整张海报不会让读者觉得在被推销。

## 九、文案类型分类风格指南

> 以下从AW历史发布文案中提炼，涵盖6大文案类型，每种类型有独立的调性、结构和语言规范。

### 类型A：封面发布文案（车型改装/装备搭配）

**用途**：配合实车改装图发布，展示AW装备装车效果。

**调性**：硬核、潮酷、有态度。像玩家之间的对话，不是品牌在说话。

**标题公式**：**车型 + 动作/变化 + 装备亮点**

**范例**：


| 文案               | 结构拆解         |
| ---------------- | ------------ |
| 猛犸817拒绝「裸车」玩法    | 车型 + 态度宣言    |
| iCAR V27露营「满分搭子」 | 车型 + 身份定义    |
| 50w的灯光一点都不亮？     | 痛点提问式        |
| 百万卫士 顶配态度！       | 车型 + 态度      |
| 别眨眼！蔚来ES6车顶"变形计" | 动作引导 + 车型+变化 |
| 城市露营一键解锁         | 场景 + 动作      |


**语言特征**：

- 用「」做强调标记（拒绝「裸车」、露营「满分搭子」、一键解锁）
- 短句为主，10字以内
- 常用词：满配、硬核、潮玩、C位出道、姿态王、显眼包
- 用"？"制造好奇心（50w的灯光一点都不亮？）

---

### 类型B：漫威联名文案

**用途**：漫威英雄IP联名产品发布、活动预热、倒计时、公众号长文。

**调性**：热血、英雄感、情感共鸣。把用户从"消费者"变成"英雄主角"。

**核心叙事逻辑**：**英雄梦 → 现实装备 → 你就是英雄**

**关键话术**：


| 话术                                 | 用途    |
| ---------------------------------- | ----- |
| 英雄梦，不只在电影里                         | 情感钩子  |
| AW将漫威英雄的「热血能量」，注入你的每一件自驾装备         | 价值主张  |
| 成为现实版「超级英雄出征」                      | 身份转化  |
| 不只是装备，更是情感的寄托                      | 品牌温度  |
| 哪怕你只是周末带着装备去露营的普通人，只要勇敢出发，你就是自己的英雄 | 人人皆英雄 |


**各英雄专属调性**：


| 英雄      | 调性关键词     | 核心话术                           |
| ------- | --------- | ------------------------------ |
| **钢铁侠** | 科技感、炫酷、能量 | "热美能量""能量核心""斯塔克营地""英雄战甲"      |
| **黑豹**  | 沉稳、力量、守护  | "沉静力量""瓦坎达""不张扬却自带气场""暗夜中的守护者" |
| **浩克**  | 狂野、力量、爆发  | "沉稳可靠""不可阻挡""SMASH""力量与野性的化身"  |
| **蜘蛛侠** | 灵动、轻快、活力  | "蛛丝般的灵动""飞檐走壁""随心所欲"           |
| **金刚狼** | 坚韧、可靠、不张扬 | "狼魂""不需要多言的可靠""粗糙外表下藏着细腻守护"    |


**漫威倒计时文案公式**：

- 倒计时1天：直接行动号召 + 英雄金句（"I am Iron Man."）
- 倒计时2~3天：英雄身份召唤 + 活动信息

### 类型C：新品上新文案

**用途**：新产品发布预告、上市推广。

**调性**：期待感、场景代入、解决痛点。让用户觉得"这正是我要的"。

**标题公式**：**新品预告/上新 + 产品名 + 核心场景价值**

**范例**：


| 文案                         | 结构拆解           |
| -------------------------- | -------------- |
| 新品预告｜AW漫威全新「钢铁侠」装备系列       | 新品预告 + IP + 系列 |
| 270°天幕让露营「无界」              | 产品 + 场景价值      |
| X90车载旋转攀梯｜新型号上新            | 产品 + 新型号       |
| 从"怕刮车"到"轻松登顶"原来只差一件「懂你」的装备 | 痛点对比 + 解决方案    |


**语言特征**：

- 用"从…到…"结构制造转变感（从"怕刮车"到"轻松登顶"）
- 用「」强调关键词（「懂你」「无界」「丝滑」）
- "你的专属…""你的…新起点"——第二人称拉近距离
- 结尾常有行动号召（"即刻行动""抢先体验"）

| 倒计时2天 | 期待值拉满       | "钢铁侠能量核心加载99%" |
| 倒计时1天 | 直接行动号召      | "漫威宇宙，即刻登陆！"   |


**周年庆专属话术**：


| 话术                  | 用途     |

### 类型E：公众号推文/长文案

**用途**：品牌故事、产品深度解读、活动回顾、门店故事。

**调性**：有温度、讲故事、像朋友在聊天。用"嘿，AW的玩家们"开头。

**开头公式**：**"嘿，AW的玩家们" + 回忆/共鸣钩子**

**范例**：


| 开头                                     | 钩子类型 |
| -------------------------------------- | ---- |
| 嘿，AW的户外玩家们，还记得AW全新一代车顶拓展平台M'LinkX刚亮相时… | 回忆共鸣 |
| 你有没有想过——你的车，不只是代步工具，它也可以是战甲座驾          | 想象钩子 |
| 你是不是也常常有这样的感觉？刷到专业自驾户外博主的内容…           | 身份共鸣 |


**长文结构**：

1. **情感钩子**（回忆/提问/想象）→ 2. **产品故事**（场景化描述，不是参数罗列）→ 3. **用户证言**（真实车主故事）→ 4. **品牌温度**（"AW始终相信…"）→ 5. **行动号召**（购买/到店/参与活动）

**品牌温度金句**（可在长文结尾反复使用）：

- "AW始终相信，好的户外装备，是有温度的。它不是功能的堆叠，更是情感的寄托。"
- "AW，不止于装备，更在于连接。连接车与户外，连接人与生活，连接每个向往户外生活的你。"
- "你不需要成为英雄，但你可以拥有英雄般的可靠。"

### 类型F：门店/渠道文案

**用途**：线下门店开业、主理人故事、渠道拓展。

**调性**：真实、有温度、本地化。像在讲一个身边人的故事。

**范例**：


| 文案                      | 结构拆解           |
| ----------------------- | -------------- |
| AW in宁波｜年轻人的户外自驾自由从这开始！ | 地点 + 人群 + 价值主张 |
| 从"头疼行李"到种草AW            | 痛点转变           |
| 80+门店，不是复制，而是"生活镜像"的生長  | 渠道规模 + 差异化理念   |


**语言特征**：

- 用真实车主故事（淘车强：从二手车老板→AW宁波主理人）

## 十二、分发与自检（可选）：与 dbs 内容技能协同

> 本章节供 **Cursor Agent / 内容同事** 在已遵守 **《AW_品牌共通基石》**（含原第一～三章 + **原第十章**）与 **本文第四～九章执行正文** 的前提下使用。涉及的 **dbs-*** 为独立技能库（dontbesilent 商业工具箱中的内容向模块），**不内联**其全文公式库，避免与本文档双源维护。

### 12.1 总原则（优先级）

1. **AW 本规范永远优先**：《共通基石》中的调性禁区与 **4.5 六条文案红线**、本文类型 A～F、漫威单独宇宙等，**不得**被 dbs 的流程替代或冲淡。
2. **dbs 为「分发形态 + 卫生与结构」补强**：仅在任务明确涉及小红书标题、短视频口播、整体内容诊断、或「AI 味」自查时启用。
3. **理念冲突时的裁量**：
  - **dbs-content** 默认强调「诊断怎么做、不替写」；在 AW 任务中若用户需要**直接产出**文案，仍以 **写出符合 AW 的成稿** 为主，五维诊断作为 **成稿后复盘或双轨审稿** 使用。  
  - **dbs-ai-check** 反对把「去 AI 味」当唯一目标；AW 要求高级、克制、不像廉价套路。**裁量**：先满足 AW 调性与禁区，再用 AI 指纹检测做 **可选卫生扫描**，不因「去痕」牺牲合理品牌句式与留白。

### 12.2 何时叠加哪一项（路由简表）


| 场景                                      | 建议叠加的 skill（Cursor `~/.cursor/skills/` 下目录名） | 说明                                                                 |
| --------------------------------------- | -------------------------------------------- | ------------------------------------------------------------------ |
| 已有 AW 调性成稿，要做 **小红书标题** 或「爆款公式感」优化      | **dbs-xhs-title**                            | 在 **不违背**《共通基石》第三章调性禁区与类型 A～F 的前提下，做公式匹配与标题变体；公式库体量较大，**以该 skill 内文档为准**。 |
| **短视频口播 / 种草脚本** 开头弱、需结构诊断与多方案          | **dbs-hook**                                 | 话题 + Hook + 可信度、悬念与口播友好；与 **第五章海报三层** 是不同介质，勿混用同一套字数公式硬套口播。        |
| 选题/形式已定，需 **整体内容怎么做** 的复盘（平台、标题、冗余、落差等） | **dbs-content**                              | 五维：文字洁癖、封面/标题、表达效率、认知落差、AI 工作流建议；用于 **传播层自检**，不替代 AW 品牌叙事。         |
| 定稿前怀疑 **AI 腔过重**、需检测报告                  | **dbs-ai-check**                             | 默认只诊断；若改写，须在 **AW 调性** 下调整，而非泛人类口语化即可。                             |


### 12.3 不建议与 AW 文案主线混用

以下 dbs 模块与 **品牌文案产出** 链路弱相关，**不必**纳入 AW 日常流程：**dbs-diagnosis**（商业模式）、**dbs-benchmark**（对标）、**dbs-action** / **dbs-slowisfast**（执行与节奏）、**dbs-chatroom** / **dbs-chatroom-austrian**、**dbs-agent-migration** 等。有单点需求时由用户 **单独 @ 或说明** 即可。

### 12.4 执行顺序建议（可选路径）

```
① 先读《AW_品牌共通基石》（含创始人战略层）→ 再按本文第四～九章完成 AW 向成稿（含类型 A～F）
       ↓ 若任务含小红书标题
② 叠加 dbs-xhs-title（公式与心理触发器匹配）
       ↓ 若任务含短视频口播开头
③ 叠加 dbs-hook（素材与开头结构）
       ↓ 若需整体复盘或平台形式
④ 叠加 dbs-content 五维（摘要级即可，不必全文复制进对话）
       ↓ 若需卫生扫描
⑤ 可选：dbs-ai-check 出检测报告，再按 AW 调性微调
```
